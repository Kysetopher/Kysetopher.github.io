Case Study | Heyjinn.ai

Hypothesis

A system designed for capturing and organizing thoughts is capable of anticipating needs.

Thoughts are captured conversationally or via direct input.

Input is analyzed and a User Profile is constructed.

User needs are anticipated and resolutions are automated or suggested.

History

My collaborator on this project, Tristen Blake, has spent the last 4 years preparing for the changes we are now experiencing due to increased accessibility to AI. During this time he has been organizing his intentions and constructing a prototype for the application we are now creating. The prototype, built using Coda, was designed for his Xybernetics Consulting business. Using it He was able intake a vast amount of information, clean the data, and execute custom actions. This techology was utilized as an ERP and CRM, and to construct a rich database to train LLMs.

While the tool was extremely powerful it was not designed for consumer use. When Tristen and I began to work together in September 2023 we determined that if we wanted to provide the world with the same technology it would need to be packaged as a standalone app. For the best results we would need to begin the rigorous process of refining the features down into an attainable product.

Design

Home

A queue of potential interests and resources: Suggested articles, reminders and important goals for the day.

These are your Reveries.

Input Bar

The input bar is used to communicate with your agent or filter items in the home view.

Navigation

The user navigates through their database by selecting Reveries and viewing their contents.

Interaction

Reverie

Each Reverie contains editable information generated from the user's input. To generate this Reverie, the user may have asked their agent to explain Schr√∂dinger's Equation.

Other Reveries may contain manually entered journal entries, summarized news articles, event planning information.

Editor

Editing Reveries is designed to be seamless across devices. Users can create objects and content manually or prompt modifications via the input bar.

Architecture

HeyJinn.ai leverages LLMs and and various API integrations to automate tasks and support conversational retrieval for a user's personal database. Handling user data honorably and securely is a major branding element and a standard for our success.

Vector Database

User data, including their personal database of notes, is processed and stored in a vector database allowing the agent to retrieve information from the user via conversation.

Task Automation

API integrations are utilized to connect other services to automate tasks and access data.

Structured Outputs

Editable content is generated using structured outputs from an LLM.

Internet Query

The agent may need up to date information to properly assist the user only avialable on the internet.

Security

Utilizing Next.js with a Supabase database allows for responsive pages and advanced security.

Cross Platform Support

Accessibility on all devices to ensure the user can record their thoughts anytime.
